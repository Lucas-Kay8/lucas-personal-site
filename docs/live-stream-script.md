# 直播稿：Trae x Lucas 个人主页重构之旅 —— 如何正确使用 AI 结对编程

## 开场白 (Intro)

大家好，我是 Lucas。今天这场直播，我想跟大家分享一个非常真实的“翻车”再到“逆袭”的故事。

主题是关于我如何使用 Trae 和 BMad 方法论构建我的个人主页。

实不相瞒，这周一沈飞老师直播课的时候，我也跟着做了一个版本。但结果是——**我把它全删了**。

为什么？因为那个版本完全不符合我的预期。它只是一个没有灵魂的网页，而不是我的主页。

今天，我用同样的时间，重新走了一遍流程，结果做出了一个我非常满意、甚至超出预期的作品。我想把这中间的“关键差异”分享给你们。

---

## 第一部分：为什么周一的版本失败了？ (The Failure)

回顾周一的过程，我总结了两个核心教训：

1.  **和分析师 (Analyst) 沟通太少**：我没有深度挖掘自己的需求，只是简单给了几个指令。AI 不知道我是谁，也不知道我真正想要表达什么。
2.  **忽视了产品经理 (PM) 的 PRD**：生成 PRD 后，我没有仔细看就直接让开发去写代码了。结果就是，开发出来的东西和我脑子里的模糊概念完全对不上。

**结论**：如果连你自己都没想清楚，AI 更不可能帮你做清楚。

---

## 第二部分：重启 —— BMad 方法论的威力 (The Success Methodology)

所以，我删除了所有代码，一切清零。这一次，我严格按照 **BMad 方法论** 的角色流程来走：`Analyst` -> `PM` -> `UX` -> `RD`。

这里有几个非常关键的步骤，也是我想重点分享的“干货”。

### 1. 深度头脑风暴 (The Analyst Session)

这是最重要的一步！我呼叫了 `@analyst` (业务分析师)。

这次它没让我直接开始，而是用了一套非常专业的**头脑风暴方法论**。这里我特别想讲讲它用到的两个方法：

*   **方法一：类比思维 (Analogical Thinking)**
    *   它问我：如果把你的网站比作一个物理实体，它是什么？
    *   在这个引导下，我们确立了 **"AI 突触 / 信息中继站"** 的核心概念。
    *   **理念**：我是一个输入者（Input），汲取全球前沿 AI 信息；经过我 20 年产品经验的思考（Process）；最后输出为高价值的智慧（Output）。
    *   有了这个“灵魂”，整个网站的逻辑瞬间就通了。

*   **方法二：风格决策 (Style Selection)**
    *   我们讨论了视觉风格。一开始是“深色极客风”，后来为了更符合中国文化和亲和力，我们调整为 **"Bright Apple Style" (明亮苹果风)**。
    *   这种决策不是拍脑袋的，而是基于它对我“AI 创业者”和“践行者”身份的分析。

**感受**：分析师不仅是整理需求，更是在帮我**提炼价值**。

### 2. 严谨的 PRD 确认 (The PM Review)

接着，`@pm` (产品经理) 基于分析师的产出写了 PRD。

这次我**非常认真地阅读了 PRD**。我发现 PRD 把我们的抽象想法变成了具体的“功能列表”和“页面结构”。确认 PRD 的过程，就是我和 AI 对齐认知的过程。

### 3. 可视化原型的力量 (The UX Prototype)

这是这次最惊喜的一步。我没有直接让它写代码，而是先呼叫 `@architect` 和 UX 角色：
*   **"先给我出一个 design-prototype.html (设计原型)"**。

当我看到那个带有毛玻璃效果 (`backdrop-filter`)、大卡片布局的原型时，我对最终成品的**具象化认知**一下子就建立了。
*   **关键点**：文字描述再准确，也不如一张图来得直接。原型让我们可以基于“视觉”去沟通，而不是基于“想象”。

### 4. 标准化的开发流 (The Standard Dev Flow)

最后进入开发。有了前面的铺垫，Trae 的开发过程非常丝滑。
*   **技术栈**：纯 HTML/CSS/JS (Vanilla Stack)。这是架构师的决策，为了轻量和便于 GitHub Pages 部署。
*   **迭代流程**：后来我发现“课程体系”排版太丑，或者“书籍封面”不够立体。我没有直接改代码，而是重新走了一遍 **UX -> PM -> RD** 的流程。
    *   UX 提出“卡片化 + 3D 悬停”的设计。
    *   PM 确认需求。
    *   RD 落地代码。
    *   **结果**：新需求的落地精准度极高，几乎一次过。

## 第三部分：成果展示 (Showcase)

大家看现在这个成品（展示网页）：

1.  **视觉风格**：通透的磨砂玻璃感，蓝白灰的高级配色，完全符合我们定义的 "Bright Apple Style"。
2.  **内容呈现**：
    *   **书籍板块**：真实的 3D 书封，悬停时有细腻的浮起效果。
    *   **课程体系**：L1/L2/L3 的等级徽章，像游戏成就卡片一样清晰严谨。
    *   **商业闭环**：不仅有内容，还有公众号和企业微信的 CTA (行动号召)，把流量转化为商业机会。

这一切，都是在一个纯静态的 HTML 文件里实现的。

---

## 结语 (Conclusion)

这次重构让我深刻体会到什么是真正的 **"人机共创" (Human-AI Co-Creation)**。

*   **Lucas (我)**：负责提供 **"灵魂"** —— 审美决策、内容定义、价值导向。
*   **Trae (AI)**：负责构建 **"骨肉"** —— 代码实现、样式调试、逻辑构建。

不要指望 AI 能猜透你的心思。用好 `@analyst` 的方法论去挖掘你的想法，用好 `@pm` 的 PRD 去规范需求，用好 UX 的原型去对齐视觉。

只要流程对，结果一定不会差。

这就是我今天想分享的，关于从“翻车”到“完美落地”的全过程。谢谢大家！
